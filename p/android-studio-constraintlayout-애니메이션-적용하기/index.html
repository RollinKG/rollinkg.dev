Android Studio ConstraintLayout 애니메이션 적용하기

Android Studio / Kotlin

2020-08-05

ConstraintLayout은 플랫한 View 구조로 복잡한 레이아웃을 만들 수 있는 ViewGroup 입니다. View간 관계 , 제약조건(Constraint) 에 따라 레이아웃이 결정되다는 점에서 RelativeLayout보다 유연하고 LayoutEditor에서 사용하기 쉽습니다. 이번 포스트에서는 ConstraintLayout의 제약 정보 조건을 담고있는 ConstraintSet 객체를 이용해 런타임 중 제약 조건을 변경하고 애니메이션을 주는 방법을 다루겠습니다.

1. ConstraintSet 이란?

ConstraintSet은 ConstraintLayout에 속해있는 View의 Constraint 정보 를 담고 있는 객체입니다. 아래 코드를 통하여 원하는 ConstraintLayout에서 원하는 ConstraintSet을 가져옵니다.

var constraintSet: ConstraintSet = ConstraintSet()

constraintSet.clone(findViewById(R.id.constraintlayout))

이때 접근하는 ConstraintLayout의 자식 View들은 반드시 id를 가지고 있어야 합니다. ConstraintSet에서 제약 정보를 가져올때 id가 지정되어 있지 않으면 런타임 에러를 일으킵니다.

2. ConstraintSet 제약 조건 변경

다음은 ConstraintSet에서 내부 View의 제약조건을 update하는 예제입니다.

//example
constraintSet.setVisibility(R.id.text_1,View.VISIBLE)
constraintSet.setVerticalBias(R.id.img_1,bias.0.2f)

ConstrainSet을 사용하여 뷰를 업데이트 하는 경우, 모두 ConstraintSet을 통해 업데이트 하는 것을 추천합니다. 후에 ConstraintSet을 다시 적용하는 과정에서 전의 변경점은 덮어 씌워지기 때문입니다.

3. ConstraintSet 적용 및 Animate

원하는 작업이 끝났으면 아래 코드로 바뀐 ConstraintSet을 적용합니다.

constraintSet.applyTo(findViewById(R.id.constraintlayout))

ConstraintSet을 사용하는 주된 이유는 내부 제약조건을 변경함에도 애니메이션을 적용하기 위함입니다. View의 Constraint Top, Bottom, Left, Right 혹은 Constraint bias의 경우 다음과 같은 방법으로만 애니메이션을 적용할 수 있습니다.

val transition = AutoTransition()
transition.duration = 500
transition.interpolator = AccelerateDecelerateInterpolator()
TransitionManager.beginDelayedTransition(findViewById(R.id.constraintlayout), transition)

//apply
constraintSet.applyTo(binding.root)

위 코드와 같이 apply 전 transition으로 애니메이션을 보간하면 제약 조건 변경에 따른 레이아웃 업데이트가 부드럽게 animate되어 이루어집니다.
